テスト駆動開発の手法

  1. やることをTODOリストに書き出す(これは随時思いつき次第追加していく)
  2. 小さいテストを一つ書く
  3. 全てのテストを実行し、先程の一つだけが失敗することを確認する
  4. コンパイルエラーを通す
  5. テストを通すための最小限の変更を行い、全てのテストが成功することを確認する
  6. リファクタリングを行い重複を除去する

テスト駆動開発の心得

  • 細かいステップを踏み続けられるようにする
  • TDDの三つの戦略
    • 仮実装 … コードでまずベタ書きの値をつかい、実装を進めるに従って徐々に変数に置き換えていく
    • 明白な実装 … すぐに頭の中の実装をコードに落とす
    • 三角測量 … 既存のコードに二つ目の実例を追加する(trueを確かめるものがあればfalseを吐き出すもの、など)
